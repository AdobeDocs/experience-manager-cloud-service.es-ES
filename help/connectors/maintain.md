Mantenimiento de un conector de AEM
============================

Este artículo contiene información sobre el mantenimiento de un conector AEM y se debe leer junto con artículos sobre la [implementación](implement.md) y el [envío](submit.md) de conectores.

Incluso después del envío inicial, puede haber motivos para que un socio actualice su conector AEM, ya sea debido a una nueva versión de AEM o independiente de ella, por ejemplo, para añadir funciones o para corregir errores. Este artículo describe el proceso para ambos casos y también describe el proceso típico de un cliente para validar los conectores al actualizar AEM.

AEM aplicaciones as a Cloud Service se actualizan diariamente con parches de mantenimiento AEM, con cambios más grandes activados mensualmente durante el lanzamiento de una función. Aunque AEM actualizaciones están pensadas para que sean compatibles con versiones anteriores y, por lo tanto, no interrumpan las aplicaciones, se aconseja a los socios proveedores que validen periódicamente que sus conectores se comportan como esperarían. El acceso a un programa o entorno de AEM será determinado por el Equipo de Socios.
